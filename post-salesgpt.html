Building a Sales Enablement GPT: What Worked and What I Changed

September 12, 2025 12 min read
Share Copy Link

Sales teams need fast answers and clean proposals. The data they need is often buried in case studies, decks, and websites. I built a Sales Enablement GPT to fix that. It ingests content, understands natural language questions, and generates tailored proposal text in seconds.

The problem

Reps were spending hours searching for proof points, client stories, and product details. Content was unstructured. Even when they found it, they still had to write new proposal drafts from scratch.

Architecture choices

  • Backend. Python with Flask for a simple and reliable API.
  • Data ingestion. WordPress API for site content and AWS S3 for PowerPoint files.
  • Structure. Normalize content into clean sections for better retrieval.
  • Prompting. Natural language queries with strict format rules for outputs.

Retrieval and prompts

I mapped content to fields like industry, challenge, solution, and results. The prompt asked the model to answer with short bullet points and references. For proposal text, I added a step that personalizes a master case study with client data.

Evaluation and iteration

  • Time to answer common questions.
  • Accuracy of retrieved facts.
  • Readability of generated proposal text.

I ran A and B prompt variants on a fixed set of questions. Simple changes like shorter instructions and clear output constraints improved results. I also added checks for stale content to avoid outdated claims.

What improved

  • Search time dropped sharply.
  • Proposal creation became faster and more consistent.
  • The team used the same facts and voice, which helped brand quality.

What I would change next

  • Add a feedback button inside the tool so users can flag unclear answers.
  • Keep a leaderboard of prompts that perform best for common tasks.
  • Expand to include contract and pricing guidance with proper controls.

Takeaway

A Sales Enablement GPT works best when data is clean, prompts are constrained, and evaluation is part of the build. The result is faster sales work and stronger proposals.
